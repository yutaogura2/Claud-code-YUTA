from pathlib import Path

import yaml
from flask import Flask

ROOT = Path(__file__).resolve().parent.parent


def test_render_yaml_targets_wsgi_app():
    data = yaml.safe_load((ROOT / "render.yaml").read_text(encoding="utf-8"))
    svc = data["services"][0]
    assert svc["startCommand"].split()[0] == "gunicorn"
    assert "web.app:app" in svc["startCommand"]
    assert "$PORT" in svc["startCommand"]
    assert svc["buildCommand"] == "pip install -r requirements.txt"


def test_requirements_has_gunicorn():
    req = (ROOT / "requirements.txt").read_text(encoding="utf-8")
    assert "gunicorn" in req


def test_app_importable_as_wsgi():
    from web.app import app
    assert isinstance(app, Flask)

# 公開デプロイ手順（Render）

Flask Web UI を Render の無料枠に公開する手順。`render.yaml` により設定は
コード化済み。以下はユーザ操作（Renderアカウントが必要）。

## 手順

1. https://render.com にサインイン（GitHub 連携が簡単）。
2. ダッシュボードで **New → Blueprint** を選択。
3. リポジトリ `yutaogura2/Claud-code-YUTA` を接続・選択。
4. Render が `render.yaml` を自動検出 → **Apply** でデプロイ開始。
5. 数分後、`https://<name>.onrender.com` が発行される（ホームが開けば成功）。

## 更新

`main` に push すると Render が自動で再デプロイする。

## 無料枠の制約（重要）

- **コールドスタート**: 15分アクセスが無いとサービスが停止し、次回アクセス時に
  30〜60秒の起動待ちが発生する（502 が出たら数十秒待って再読込）。
- **キャッシュ揮発**: 無料枠はファイルが永続しないため、再起動のたびに
  `data/cache` が消え、40銘柄を再取得する（実行が遅くなる）。
- **yfinance 制限**: クラウドIPからの取得は Yahoo にレート制限/ブロックされる
  ことがあり、スクリーニングが間欠的に失敗し得る。
- 安定運用が必要なら有料枠、またはキャッシュ外部化（本リポジトリ未対応）を検討。

## ローカル実行（デプロイ不要で使う場合）

```powershell
.\run.ps1 web   # http://127.0.0.1:5000
```
```
